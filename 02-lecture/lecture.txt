1 Дерево файлів, робоча директорія

Структура каталогів Лінукс 
У ОС Лінукс відсутнє поняття логічних дисків C, D і т. ін. Всі директорії розташовані у вигляді дерева. 
Як виглядає дерево каталогів можна побачити якщо у консолі codeanywhere виконати команду ls /

Кожному користувачу у Лінукс надається його особиста директорія у директорії /home. У випадку із 
codeanywhere директорія надана користувачу cabox (від code anywhere box), а в ній налаштована для 
роботи із веб-сервером директорія workspace, в якій вже можна створювати різні директорії під різні 
проекти що будуть доступні з веб.



2 Абсолютні і відносні шляхи

2.1 Абсолютний шлях це шлях від кореня файлової системи /home/cabox/workspace. Щоб побачити його у 
консолі використовується команда pwd.

Приклад. 
Перейти за абсолютним шляхом до своєї домашньої директорії: cd /home/cabox 


2.2 Відносний шлях - шлях відносно поточної директорії. 

Приклади:
2.2.1 Перейти з поточної, домашньої директорії до робочої директорії: cd ./workspace
2.2.2 Створити нову директорію рівнем вищу за поточну: mkdir ../newdir
2.2.3 Переглянути вміст домашньої директорії з поточної: ls ../../

У відносних шляхах крапка означає поточну директорію, дві крапки директорію рівнем вищу за поточну.
Абсолютні та відносні шляхи можна використовувати у всіх командах, які потребують зазначення шляхів.



3 Видалення

Для видалення файлів використовується команда rm, а для видалення директорії команда rm -r.
Команда rm -r видаляє й файли. 

Для видалення треба ввести команду rm -r /шлях/до/директорії і натиснути Enter.

Приклади:
3.1 Видалити все з поточної директорії workspace.
Для того щоб видалити все з поточної директорії workspace треба пересвідчитись що ми знаходимось саме
в ній. Для цього можна скористатись командою pwd, або просто подивитись на назву директорії що знаходиться
перед символом $ у командному рядку.  
Якщо ми знаходимось саме у директорії workspace то слід скористатись командою rm -r ./* де зірка означає
всі файли і директорії.

3.2 Видалити директорію newdir що знаходиться рівнем вище від workspace.
Для видалення використовуємо команду rm -r ../newdir

Перейти до директорії workspace і клонувати туди git clone https://github.com/konotopolytech/web-course.git

вмістом.


5.1 Що таке html

HTML означає мова гипертекстової розмітки (Hyper Text Markup Language) для створення веб сторінок.
HTML описує структуру веб сторінок використовуючи розмітку.
HTML елементи це будівельні блоки для створення сторінок.
HTML елементи називаються тегами.
HTML складається з елементів вмісту, таких як "заголовок", "параграф", "таблиця" тощо.
Браузер не відображає теги, але використовує їх для відтворення вмісту сторінок.


5.2 Тегі
<!DOCTYPE html> декларує що документ буде у форматі HTML5
<html> кореневий елемент HTML сторінки
<head> елемент що містить мета-дані які описують властивості поточного документу
<title> визначає назву HTML сторінки. Міститься у <head>.
<body> елемент який містить в собі видиму частину документу
<h1> елемент визначає найбільший заголовок на сторінці. Існують елементи від <h1> до <h6>.
<p> елемент визначає параграф, зазвичай блок тексту.
<a> елемент гіперпосилання містить назву та адресу ресурсу на який він посилається.
<img> елемент які вбудовує до сторінки зображення.

HTML тегі зазвичай є парними на кшталт <p> та </p>.
Перший тег зветься відкриваючим, другий закриваючим. Закриваючий тег відрізняється від відкриваючого
наявністю правого слешу / який ставиться одразу після символу <


5.3 Атрибути тегів

Атрибути забезпечують додаткову інформацію про HTML елемент.

Всі HTML елементи можуть мати атрибути.
Атрибути завжди додаються до відкриваючого тегу.
Атрибути зазвичай мають вигляд пари імені атрибуту і його значення: name="value".

Приклад атрибутів href i target:
<a href="https://google.com" target="_blank" >Google</a>


5.4 Практичне заняття

Перейти до директорії html-study (клон репозиторія) і створити у ньому файл index.html в якому побудувати
структуру документу використавши тегі <!DOCTYPE html>, <html>, <head>, <title>, <body>.

Додати на сторінку назву "Перша html-сторінка", створити два параграфи у першому написати текст: 
"На курсах з основ веб-розробки слухачі набувають навичок зі створення сторінок засобами мови html".

У другому параграфі написати: "Курс проходить на базі Політехнічного технікуму КіСумДУ". Слова 
"Політехнічного технікуму КіСумДУ" треба включити до гіперпосилання, яке б вело на головну сторінку
http://kpt.sumdu.edu.ua/ головну сторінку сайту технікуму.


6 Збереження наробок у репозиторії


Для додавання у репозиторій у директорії html-study слід виконати команду git add . яка включить всі 
нові та змінені файли до індексу git.

Після цього виконати команду git commit -m "Стислий опис дій виконаних із файлами". Таким чином всі
зміни будуть збережені у локальному сховищі.

Наступним кроком слід відправити всі нові зміни до віддаленого репозиторію. Для цього існує команда
git push
