1 Namespaces

У PHP простори імен використовуються для вирішення двох основних проблем:
1) Запобігання конфлікту між кодом розробника і внутрішніми класами/функціями/константами PHP.
2) Можливість створювати псевдоними для дуже довгих імен і покращити читабельність коду.

Приклади
1.1 Декларація двох класів з однаковою назвою у різних просторах імен.
1.2 Декларація двох класів з різними назвами у різних просторах імен - незручність підключення.


2 Composer & Composer Autoload

Composer менеджер пакунків, який дозволяє втсановлювати і фіксувати залежності, для повторного 
використання.

Крім того завдяки Composer можна налаштувати автоматичне підключення класів до проекту.
Приклад composer.json
{
  // Завантаження залежності  
  "require": {
    "ehime/hello-world": "1.0.5"    
  },
  // Автопідключення
  "autoload": {
    "psr-4": {
        "App\\" : "app/"
    }
  }
}
