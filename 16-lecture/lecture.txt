1 Встановлення Yii Framework

Для встановлення Yii можна використати composer. Приклад команди встановлення з оф. сайту фреймворку:

composer create-project --prefer-dist yiisoft/yii2-app-basic basic

В результаті виконання команди фреймворк буде встановлений у директорію basic. Назву директорії у яку
виконується встановлення можна змінювати.

Інші можливі способи встановлення фреймворку представлені на оф. сайті за адресою 
https://www.yiiframework.com/doc/guide/2.0/ru/start-installation


2 Запуск фреймворку

Для того, щоб фреймворк запускався у контейнері codeanywhere заумовчанням треба виконати наступні дії

а) Створити файли .htaccess у директорії фреймворку та у директорії web. 

б) До цих файлів слід помістити код зі сторінки: 
https://www.yiiframework.com/wiki/766/easy-way-to-redirect-httplocalhostyii2-basic-to-httplocalhostyii2-basicwebindex-php

в) Після цього у директорії workspace слід створити файл index.php до якого помістити код
<?php
header("location: /basic/web/");
?>
Інформація про функцію header доступна на сторінці http://php.net/manual/ru/function.header.php


3 Структура фреймворку

basic/                  кореневий каталог застосунку
    composer.json       використовується Composer'ом, містить опис застосунку
    config/             конфигураційні файли
        console.php     конфигурація консольного застосунку
        web.php         конфигурація Web застосунку
    commands/           містить класи консольних команд
    controllers/        контролери
    models/             моделі
    runtime/            файли, які генерирує Yii під час виконання застосунку (логі, кеш і т.ін.)
    vendor/             містить пакунки Composer'а й, власне, сам фреймворк Yii
    views/              відображення застосунку
    web/                коренева директорія Web застосунку. Містить файли, які доступні через Web
        assets/         скрипти, що використовуються застосунком (js, css)
        index.php       точка входу у застосунок Yii. З нього починаєтся виконання застосунку
    yii                 скрипт виконання консольного застосунку Yii


4 Патерн MVC (Model-View-Controller)

Схема за якою працює фреймворк доступна за адресою:
https://www.yiiframework.com/doc/guide/2.0/ru/images/application-structure.png

У кожному застосунку Yii є точка входу у застосунок, web/index.php це єдиний PHP-скрипт доступний 
для виконання з Web. Він прийимає вхідний запит і створює екземпляр застосунку. Застосунок обробляє 
вхідні запити з допомогою компонентів і відправляє запит контролеру. Віджети використовуються у 
Виглядах для будування динамічних інтерфейсів сайту.


Життєвий цикл користувацького запиту

Користувач звертається до точки входу web/index.php.
Скрипт завантажує конфігурацію configuration та створює екземпляр застосунку для подальшої обробки 
запиту.
Застосунок визначає маршрут запиту із допомогою компоненту застосунку "запрос".
Застосунок створює екземпляр контролеру для виконання запиту.
Контролер, у свою чергу, створює дію и застосовує для неї фільтри.
Якщо хоч би один фільтр дасть збій, виконання застосунку зупиняється.
Якщо всі фільтри пройдено - застосунок выконується.
Дія завантажує модель даних. Найвирогідніше за все з бази даних.
Дія генерує вигляд, відображая в ньому дані (у т.ч. отримані з моделі).
Згенерований вигляд застосунку передається як компонент "відповідь".
Компонент "відповідь" відправляє готовий результат роботи застосунку браузеру користувача.

https://www.yiiframework.com/doc/guide/2.0/ru/images/request-lifecycle.png


5 Створення першого action та відображая

У контролері SiteController.php створити наступну функцію
public function actionSay($message = 'Hello') 
{
    return $this->render('say', ['message' => $message]);
}

У директорії views створити файл say.php з наступним вмістом
<?php
use yii\helpers\Html;
?>

<?= Html::encode($message)?>
